# 릴리스 워크플로우에서 npm 버전 고정 규칙
- `pnpm add -g npm@latest` 대신, 특정 안정된 npm 버전(예: 8.19.3)으로 고정하여 사용한다.
- npm 버전은 `.npmrc` 또는 환경 변수로 명시적으로 지정한다.
- 릴리스 관련 워크플로우에서는 npm 버전 변경이 필요할 경우, 버전 업데이트 후 테스트를 반드시 수행한다.
- npm 버전 고정은 재현 가능성과 안정성을 위해 필수적임을 인지한다.
- 버전 변경 시, 변경 내역을 문서화하고 관련 테스트를 수행하여 릴리스 안정성을 확보한다.
