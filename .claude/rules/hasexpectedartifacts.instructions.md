# `hasExpectedArtifacts()` 함수 검증 강화 규칙
- `hasExpectedArtifacts()`는 출력 디렉토리의 존재 여부만 확인하므로, 내부 파일의 상태(존재 여부 및 무결성)도 함께 검사하도록 개선한다.
- 내부 파일이 삭제되거나 손상된 경우를 감지할 수 있도록, 파일 존재 여부뿐 아니라 파일 크기 또는 해시값 검증을 수행하는 유틸리티를 활용한다.
- 스크립트 내에서 `fs` 또는 `crypto` 모듈을 이용하여 파일 무결성 검사를 수행하는 함수를 추가한다.
- 이 규칙은 해당 경로 내 코드 생성 신뢰성을 높이기 위해서만 적용하며, 다른 경로나 범위에는 영향을 주지 않는다.
- 검증 로직은 테스트 또는 빌드 과정에서 호출되어, 캐시 히트 여부를 보다 정확하게 판단할 수 있도록 한다.
