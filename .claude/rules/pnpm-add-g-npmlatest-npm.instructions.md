# 릴리스 워크플로우에서 npm 버전 고정 규칙
- `pnpm add -g npm@latest`와 같은 명령어 사용을 피하고, 특정 안정된 npm 버전으로 고정할 것
- npm 버전은 `.npmrc` 또는 워크플로우 환경 변수에 명시하여 결정성을 유지할 것
- 릴리스 관련 스크립트에서는 항상 고정된 npm 버전으로 실행되도록 설정할 것
- 도구 체인 버전 관리를 위해 `package.json` 또는 별도 버전 관리 파일에 명시할 것
- 변경 시에는 반드시 버전 업데이트 후 테스트를 수행하여 재현 가능성을 확보할 것

# 릴리스 워크플로우에서 npm 버전 관리 규칙
- `pnpm add -g npm@latest` 대신, 안정적인 특정 npm 버전(예: 8.19.3)을 명시적으로 사용하거나, 프로젝트에 번들된 npm 버전을 활용하세요.
- npm 버전 고정을 위해 `.npmrc` 또는 `package.json`의 `engines` 필드를 활용하여 일관성을 유지하세요.
- 릴리스 관련 GitHub 워크플로우에서는 npm 버전 변경이 재현 가능성을 해치지 않도록 주의하세요.
- npm 버전 업데이트 시, 사전에 충분한 테스트를 수행하여 예기치 않은 문제를 방지하세요.
- 버전 고정된 npm을 사용하는 경우, 정기적으로 버전 업데이트 후 호환성 검증을 수행하세요.
