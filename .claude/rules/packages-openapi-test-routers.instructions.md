# 인터셉터 루프 내 에러 체인 전달 규칙
- 변환된 에러를 체인으로 전달하여 후속 인터셉터가 변경된 에러를 인지할 수 있도록 하세요.
- 각 인터셉터에서 원본 `error` 대신, 변환된 에러를 다음으로 넘기도록 `next(error)` 호출 방식을 수정하세요.
- `error` 객체를 재사용하거나 새로 생성하는 패턴을 활용하여 에러 체인을 유지하세요.
- `middleware` 또는 `interceptor` 패턴을 지원하는 유틸리티를 활용하여 에러 전달 로직을 표준화하세요.
- 에러 체인 유지 여부를 테스트하는 유닛 테스트를 추가하여 규칙 준수 여부를 검증하세요.

# 인터셉터 에러 체인 전달 규칙
- 인터셉터 내에서 변환된 에러는 반드시 다음 인터셉터에 전달하기 위해 체인으로 연결해야 합니다.
- 각 인터셉터는 이전 단계에서 변환된 에러를 인수로 받아야 하며, 원본 에러 대신 변환된 에러를 전달해야 합니다.
- 에러 체인 전달 시, `next(error)` 또는 유사한 패턴을 사용하여 변환된 에러를 넘겨야 합니다.
- 에러 변환 후, 원본 에러 대신 변환된 에러를 계속해서 전달하는 일관된 패턴을 유지하세요.
- 이 규칙은 오류 처리의 일관성과 디버깅 용이성을 위해 필수적입니다.
